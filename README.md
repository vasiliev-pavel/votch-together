# 🍿 Votch Together

Совместный (синхронный) просмотр видео в браузере. Один человек открывает видео и
отправляет другому ссылку-приглашение — дальше play / pause / перемотка
синхронизируются у обоих. Чат прилагается.

**Без бэкенда.** Синхронизация идёт напрямую между браузерами через WebRTC
(PeerJS + бесплатный публичный сигнальный сервер). Поэтому это обычный статичный
сайт — его можно положить на GitHub Pages или Vercel как есть.

## Что поддерживается

- ✅ **YouTube** — через официальный IFrame Player API.
- ✅ **Прямые ссылки на видеофайлы** (`.mp4`, `.webm`, `.ogg` и т.п.) по `https`.
- ✅ **Демонстрация экрана** — для любого другого сайта. Один человек шарит
  вкладку с видео через WebRTC (`getDisplayMedia`), второй смотрит трансляцию.
- ❌ Встроить произвольный «сайт с видео» в окно и синхронизировать его плеер
  нельзя: почти все сайты запрещают встраивание в iframe (`X-Frame-Options`) и
  не дают управлять плеером из-за cross-origin. Это ограничение браузеров —
  поэтому для сторонних сайтов используется демонстрация экрана.

### Демонстрация экрана

1. Дождитесь, пока друг подключится к комнате.
2. Нажмите «🖥 Поделиться экраном» и выберите вкладку с видео.
   В Chrome поставьте галочку «Сделать доступным звук вкладки», чтобы был звук.
3. Друг видит вашу трансляцию в плеере. Нажмите «⏹ Остановить» для завершения.

> Работает с любым видимым сайтом/приложением. **Исключение:** контент с DRM
> (Netflix, Disney+ и т.п.) при захвате экрана часто чернеет — это защита
> платформы, обойти её нельзя. Качество трансляции зависит от интернета того,
> кто делится.

## Как пользоваться

1. Открой сайт — ты автоматически становишься **ведущим**.
2. Вставь ссылку на видео, нажми «Открыть».
3. Скопируй ссылку-приглашение и отправь другу.
4. Друг открывает ссылку — подключение P2P, и вы смотрите синхронно.

> Управлять (play/pause/перемотка) могут оба. Ведущий раз в несколько секунд
> рассылает «маяк» времени, чтобы убрать рассинхрон.

## Локальный запуск

Нужен любой статичный сервер (просто открыть файл по `file://` не даст работать
WebRTC/clipboard):

```bash
# Python
python -m http.server 8000
# или Node
npx serve .
```

Затем открой <http://localhost:8000>. Для теста на одном компьютере открой
ссылку-приглашение во втором окне/вкладке.

## Деплой

### GitHub Pages

1. Создай репозиторий и запушь эти файлы в ветку `main`.
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch.**
3. Ветка `main`, папка `/ (root)` → **Save**.
4. Через минуту сайт доступен по `https://<username>.github.io/<repo>/`.

```bash
git init
git add .
git commit -m "Votch Together"
git branch -M main
git remote add origin https://github.com/<username>/<repo>.git
git push -u origin main
```

### Vercel

1. Импортируй репозиторий на <https://vercel.com/new>.
2. Framework Preset: **Other**, Build Command — пусто, Output Directory — `.`.
3. Deploy. Готово.

Либо через CLI:

```bash
npm i -g vercel
vercel
```

## Замечания

- Сайт должен открываться по **https** (GitHub Pages и Vercel дают его сразу) —
  иначе WebRTC и доступ к буферу обмена будут ограничены.
- Прямые видео тоже должны быть по `https`, иначе браузер заблокирует mixed content.
- Некоторые ролики на YouTube запрещают встраивание — тогда плеер покажет ошибку,
  это ограничение конкретного видео.
- Публичный сигнальный сервер PeerJS бесплатный и иногда бывает занят. Для
  стабильности можно поднять свой [PeerServer](https://github.com/peers/peerjs-server)
  и передать его в `new Peer(undefined, { host, port, path })` в `app.js`.
